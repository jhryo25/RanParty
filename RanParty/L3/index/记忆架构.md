## 数据流向

```
用户输入
  │
  ▼
┌─────────────────────────────────────────────────────────┐
│                     SuperCat 对话中枢                    │
│                                                         │
│  ① 注入 L0（SOUL + AGENTS + TOOL）       ← 每次必加载    │
│  ② 注入 L1（HUB.md）                      ← 首次加载      │
│  ③ 恢复上轮对话（SuperCat_active.txt）    ← 断点续传      │
│  ④ 按需加载 L2（Skill/Exp）              ← 关键词触发    │
│  ⑤ 生成回复 + 调用工具                                    │
│                                                         │
│  收工信号                                                │
│     │                                                    │
│     ▼                                                    │
│  ⑥ 写入 L3（index / project / history）  ← 知识沉淀      │
│  ⑦ 更新 L2（版本号+1）                                   │
│  ⑧ 更新 HUB（目录树+版本）                               │
│  ⑨ 清理 CatTemp                                          │
│  ⑩ 写入 SuperCat_active.txt             ← 会话存档      │
│  ⑪ API 调用留档 CALL-*.txt                               │
└─────────────────────────────────────────────────────────┘
```

## 短期记忆 vs 长期记忆 对照表

| 维度 | 🔵 短期记忆 | 🟢 长期记忆 |
|------|------------|------------|
| **位置** | 进程内存 + API 上下文窗口 | 磁盘文件（RanParty/ Config/ Log/） |
| **生命周期** | 单次会话，窗口关闭即释放 | 持久化，跨会话保留 |
| **内容性质** | 对话流、工具调用链、帧状态 | 知识资产、项目档案、经验沉淀 |
| **容量限制** | API 上下文窗口上限 | 无硬限制（文件系统容量） |
| **检索方式** | 自然语言上下文内引用 | 文件名搜索 / 锚点读取 / glob 查找 |
| **写入者** | AI 回答 + 工具返回结果 | 收工流程手动沉淀 |
| **遗忘风险** | 高（超出窗口即丢失） | 低（除非手动删除） |
| **加载方式** | L0 全量注入，L2 按需 | 按需 file_read_between / file_find |
| **代表文件** | SuperCat_active.txt (恢复点) | L3/index/ L3/project/ L3/history/ |
| **互转关系** | 会话中有价值片段 → 收工沉淀 → | → 写入 L3 成为长期记忆 |

## 四级加载时序（单次会话启动）

```
时间 ─────────────────────────────────────────────────►

t0: RanParty.exe 启动
│
├─ t1: 读取 Config/config.cfg          ← 长期记忆
├─ t2: C_CatRegistry 初始化
│
├─ t3: 注入 L0 指令集                   ← 短期记忆注入
│      ├─ SOUL.md      （身份锚点）
│      ├─ AGENTS.md    （运行红线）
│      └─ TOOL.md      （工具指南）
│
├─ t4: 注入 L1 中枢                     ← 短期记忆注入
│      └─ HUB.md       （路由 + 收工流程）
│
├─ t5: 恢复 SuperCat_active.txt        ← 长期记忆恢复
│      （若存在 → 上轮对话续接）
│
├─ t6: 进入 S_Idle，等待用户输入
│
│   ┌─ 用户消息 ──────────────────────┐
│   │                                  │
│   │  t7: 关键词匹配 L2 按需加载       │  ← 短期/长期混合
│   │       ├─ C# 关键词 → CSharp.md   │
│   │       ├─ AVG 关键词 → AVGEngine  │
│   │       └─ ...                     │
│   │                                  │
│   │  t8: SuperCat 推理 → 回复/工具调用│
│   │                                  │
│   │  t9: 工具操作读写 L3/文件          │  ← 长期记忆变更
│   │                                  │
│   │  t10: 回合结束                    │
│   │       ├─ 上下文消息追加到窗口     │   ← 短期记忆增长
│   │       └─ 持久化 SuperCat_active   │   ← 长期记忆存档
│   └──────────────────────────────────┘
│
├─ t11: 收工/沉淀                       ← 短期 → 长期转换
│        ├─ CatTemp 写草稿
│        ├─ 确认后写入 L3/index/ 或 L3/project/
│        └─ 更新 HUB.md 版本号
│
└─ t12: 退出
         ├─ 归档 session.txt + CALL-*.txt
         └─ 进程内存释放（短期记忆消失）
```

> **关键转换节点：** 收工流程是将「短期记忆中的有价值片段」萃取为「长期记忆资产」的唯一通道。没有收工，会话内容随窗口关闭而消失。
