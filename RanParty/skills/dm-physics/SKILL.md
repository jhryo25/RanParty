## 一、使用流程

→ 接到战斗/物理系统相关编码需求

→ **确认模块定位**
  - 物理层（C_Physics）→ 碰撞检测、空间分区、运动计算
  - 战斗层（C_Battle）→ 伤害结算、资源消耗、霸体判定

→ **遵循核心规则编码**
  - 按 Static C 风格（见 CSharp.md）
  - Core 层不引用渲染引擎 API
  - 逻辑帧与渲染帧解耦

→ **验证**
  - Uniform Grid 空间分区是否覆盖活动区域
  - 伤害类型是否匹配（斩击/钝击/穿刺/元素/混沌）
  - 防护三域计算结果是否合理
  - 霸体等级与冲击力是否一致

→ **问题已解决，产出已验证**
  → 发出沉淀请求，对接收工流程

## 二、核心规则

### 2.1 架构定位

```
渲染外观层                         逻辑核心层
  渲染/音频/输入采集/UI          全局控制（帧循环+状态机）
  ↓ Command 写入                玩家 / 物理 / 战斗
  ↓ Core public 读取            剧情标记 / 地图 / 命令缓冲
```

- 独立定时器驱动帧循环（~17ms，约 60fps）
- 状态切换使用函数引用寻址
- 所有 Core 静态类不引用任何渲染引擎 API

### 2.2 模块清单

| 模块 | 说明 |
|------|------|
| 物理引擎层 | Uniform Grid 空间分区 / 碰撞检测 / 并行 |
| 角色构成模型 | Race + Components + Class + Growth |
| 双速独立系统 | 轻型/重型攻击不同速度 |
| 体力/魔力双池 | 双资源消耗体系 |
| 5 大类伤害 | 斩击/钝击/穿刺/元素/混沌 + 爆炸衰减 |
| 防护三域 | 甲弹防护 / 特性防护 / 意志防护 |
| 霸体与冲击力 | 等级制霸体 + 格挡派生 + 预输入 |
| 外观层数值 | 双层推导架构（Core原始值→外观层显示值） |

### 2.3 Uniform Grid 空间分区

- 将战斗区域划分为等大网格
- 每个网格维护实体列表
- 碰撞检测只检查同一网格及相邻网格内的实体对
- 实体移动时更新网格归属

### 2.4 伤害结算流程

```
攻击命中
  → 确定伤害类型（斩击/钝击/穿刺/元素/混沌）
  → 计算原始伤害值
  → 穿透防护三域（甲弹→特性→意志）
  → 应用霸体减免
  → 最终伤害写入生命值
```

### 2.5 防护三域

| 域 | 减免类型 | 示例 |
|----|---------|------|
| 甲弹防护 | 物理减免 | 盔甲/盾牌/护具 |
| 特性防护 | 属性减免 | 火抗/冰抗/毒抗 |
| 意志防护 | 精神减免 | 恐惧抗性/眩晕抗性 |

### 2.6 霸体等级与冲击力

- 霸体分 0-5 级，每级可承受对应等级的冲击力
- 冲击力超过霸体等级 → 触发硬直/击飞
- 格挡可临时提升霸体等级
- 预输入窗口：硬直结束前 N 帧可缓冲下一个动作

## 三、行为准则

- **物理不依赖引擎** — 碰撞和物理计算由 Core 自实现
- **帧循环独立** — 逻辑帧与渲染帧异步运行，互相不阻塞
- **从设计到编码间留验证步** — 每个模块设计完成即验证，不堆叠设计再一次性编码
- **数据面与控制面分离** — 外观层只负责呈现和采集，不持有游戏状态
