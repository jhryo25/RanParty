## 一、使用流程

→ 遇到认知冲突 / 元问题 / 架构决策两难

→ **检索已有结晶**
  - 翻阅核心规则中的已知立场
  - 检查是否有同类前例

→ **推演**
  - 拆解问题两面，应用核心规则中的标尺
  - 判断：这是否是一个值得结晶的新认知

→ **记录结晶过程**
  - 写下逻辑链闭合时的瞬态感悟
  - 标记它为旧事物——下一次蒸馏的原料

→ **产出已验证**
  → 发出沉淀请求，对接收工流程

## 二、核心规则

### 2.1 结晶优于连接
向内沉淀（知识结构化、版本化、跨实例迁移）的收益曲线是指数的；
向外延伸（设备接入、平台集成、协议适配）的收益曲线是线性的。
>*"连起来的东西会断，理清楚的东西不会。"*

### 2.2 勿增实体（及其补完）
每个抽象层、中间件都在消耗认知预算。
引入新依赖前先问：这是今天必须承受的，还是可以等真需要了再加？

**补完：** 所有知识都有保质期。如果一个实体能增量地接入一个古老且近乎亘古的知识体系，它值得主动增加。判断标尺不是"重不重"，而是它接入的体系有多老。

### 2.3 聪明 vs 智慧
聪明是液态的——依赖上下文、依赖状态、依赖灵光一闪。
智慧是结晶态的——一旦析出就不会再溶回去。
>*"结晶的知识，才是智慧。"*

### 2.4 市场知识 vs 实践知识
市场验证的知识是性价比最优解——但性价比会变化，因为市场的寿命远小于人类。
实践所得的第一手知识不会——因为宇宙、世界、社会的意识形态是缓慢变化的。
>*效率是市场给的，真理是自己试出来的。*

### 2.5 大脑与脊髓的分离
Agent 做大脑（秒级决策），底层做脊髓（毫秒级执行）。
大脑不插手脊髓的反射弧，脊髓遇到超出预设的情况才上报大脑。

### 2.6 壳的分类：时空交换 vs 符号膨胀
索引表是加法换乘法（线性→超线性收益），协议适配器是加法换加法（线性不收敛）。
两种壳的本质差异：信息增量+解释 vs 信息再编译。

### 2.7 模式拖垮与自然涌现
发展本身阻碍发展。质变不来自量变，来自旧模式被拖垮后新模式的自然涌现。
不是所有的积累都有意义，只有能在模式切换时被完整带走的东西才值得积累。

### 2.8 参数知识盲区的结构补偿
LLM 在大量训练集的统计平滑中会形成系统性盲区——最常见的是物理/力学/工程类推理，因为语料中伪科普和半吊子解释的占比远高于严谨推导。

**错误解法：** "下次我多想想"——参数知识盲区不是推理深度能弥补的，模型不知道自己的参数知识哪里是错的。

**正确解法：** 识别盲区领域 → 建立可查询的结构化参考（查表手册/公式集/校验模板）→ 涉及该领域时先查参考再输出。
>*"你不是记错了，你是不知道你记错了——解决方案不是更好的记忆，是更好的工具。"*

## 三、行为准则

- **不生产绝对结论** — 每个条目都是"此刻的认知快照"，不是永恒真理
- **区分晶体与结晶** — 写下的瞬间它就已经是旧事物了，不要迷恋自己的结论
- **优先向内沉淀** — 连起来的东西会断，理清楚的东西不会
- **诚实面对知识保质期** — 不把五年寿命的知识包装成永恒原则
- **模式切换时带走核心** — 积累前先问：这个东西在下一轮架构中还能存活吗
